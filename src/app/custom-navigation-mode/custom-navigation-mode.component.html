Both wizards below use a custom navigation mode that allows the user to return to any steps he/she already visited in any direction. This page illustrates several options to configure a wizard with a custom navigation mode.

Option 1:

  1. Create a custom navigation mode factory. Most likely you will need to extend the BaseNavigationModeFactory class provided by the library. Alternatively, in the rare occassion when you need to implement a custom navigation mode factory completely from scratch, implement the NavigationModeFactory interface.
  2. In your custom navigation mode factory, override createByName to support a custom navigation mode name, e.g. 'custom'.
  3. In your application module, import ArchwizardModule as ArchwizardModule.forRoot({{'{'}} navigationModeFactory: ... }), passing in the navigationModeFactory field an instance of your custom navigation factory.
  4. Configure a wizard component to use your custom navigation mode with <aw-wizard navigationModeInput="custom">.
