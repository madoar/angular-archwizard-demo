At first blush, no one would prefer the second version over the first because it is so verbose. But it has its advantages. To summarize:

Pros:
More declarative and requires less knowledge of wizard's internal DOM structure
Makes it easy to completely redefine the global color theme, including all layouts, indicator styles and hover states
Does not require flimsy techniques such as !important to achieve proper application of style rules
Cons:
More verbose for simple cases
Still requires some knowledge of angular-archwizard operation and wizard states. For an example, see 'editing' state definitions above
SCSS variables do not cover everything, so for ultimate flexibility you might still need custom CSS
